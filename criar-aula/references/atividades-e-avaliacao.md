# Atividades e Avaliacao

## Quando Criar Atividade

Criar atividade quando:

- o tema exige aplicacao;
- a aula e de revisao ou consolidacao;
- o usuario pede AC, N1, N2, quiz, ficha ou case;
- ha risco de a aula ficar expositiva demais.

## Estrutura de Case

Todo case deve ter:

1. Contexto organizacional.
2. Tensao ou problema.
3. Dados suficientes para diagnostico.
4. Perguntas.
5. Resolucao esperada.
6. Criterio de correcao se avaliativo.

## Tipos de Pergunta

### Diagnostico

Pergunta: "O que esta acontecendo?"

Usar para identificar falha, pilar fraco, degrau ou risco.

### Decisao

Pergunta: "O que voce faria primeiro?"

Usar para avaliar prioridade e raciocinio de gestao.

### Justificativa

Pergunta: "Por que?"

Usar para impedir resposta decorada.

### Sintese

Pergunta: "Explique em uma frase."

Usar para fechamento ou avaliacao curta.

## Resolucao Esperada

Toda pergunta importante deve ter uma resposta esperada.

Formato recomendado:

```text
Resposta forte:
- diagnostica corretamente;
- usa conceito da aula;
- justifica com evidencia do case;
- propoe acao concreta.

Resposta parcial:
- identifica parte do problema, mas nao conecta ao conceito;
- propoe solucao generica.

Resposta fraca:
- culpa apenas a tecnologia;
- nao usa evidencia do case;
- nao propoe acao.
```

## Rubrica Simples

Para atividades curtas:

- 40% diagnostico correto;
- 30% justificativa com conceito;
- 20% proposta concreta;
- 10% clareza.

Adaptar pesos conforme a avaliacao.

## Dinamicas

Opcoes:

- individual com recolhimento;
- dupla com debate rapido;
- grupos com apresentacao relampago;
- ficha diagnostica;
- matriz comparativa;
- estudo de caso com resolucao comentada.

Sempre indicar tempo sugerido quando for material de aula.
